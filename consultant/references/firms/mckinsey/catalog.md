# McKinsey Framework Catalog

Updated: 2026-03-03
Companion: [process.md](process.md) — 4-tier stack workflow and quality gates.

## Purpose

This companion file captures what each framework/tool/strategy is about, beyond workflow steps:

- What it is.
- Why and when it is used.
- Inputs required.
- Outputs produced.
- How it is typically shown in executive decks.
- What decision it is meant to unlock.

Naming note: framework labels reflect internal phrasing and may differ from public taxonomy.

## Tier 1: Diagnostic Frameworks

### Issue Tree

What it is
- A structured problem decomposition system that turns a broad executive problem into testable sub-problems and hypotheses.

Why/when used
- Use first on most strategy and transformation cases where problem scope is unclear.

Inputs
- Sponsor objective, baseline performance, timeline pressure, known constraints.

Outputs
- Problem statement.
- MECE decomposition tree.
- Hypothesis tracker by leaf node.

Deck expression
- Issue tree overview page + hypothesis tracker pages.

Decision unlocked
- What analytical workstreams exist, who owns them, and where to allocate team effort first.

### CEO Agenda Alignment Overlay

What it is
- A prioritization overlay that maps analytical workstreams against sponsor priorities.

Why/when used
- Use when analytical priority and sponsor attention are misaligned or politically sensitive.

Inputs
- Sponsor interview output, issue-tree workstreams, preliminary impact estimates.

Outputs
- Priority alignment matrix.
- Resource reweighting guidance.

Deck expression
- Impact vs alignment quadrant with workstream bubbles.

Decision unlocked
- Which streams to accelerate, educate upward on, or deprioritize.

## Tier 2: Analytical Engines

### Benchmark Analysis

What it is
- A standardized method to compare client performance to peers/best-in-class and convert performance gaps into value-at-stake.

Why/when used
- Use whenever the case requires external performance context or ambition-setting.

Inputs
- Client KPIs, comparable peer set, normalization assumptions.

Outputs
- Normalized gap table.
- Gap-to-value translation.
- Prioritized opportunities by value and feasibility.

Deck expression
- Comparison visuals and benchmark bridge/waterfall.

Decision unlocked
- Which performance gaps matter financially enough to act on now.

### Value Driver Tree Quantification

What it is
- A mathematically linked decomposition from top KPI to operational drivers, with sensitivity to quantify leverage.

Why/when used
- Use when leadership asks "which levers move the number most".

Inputs
- Root KPI definition, formula logic, operational metrics, baseline and benchmark values.

Outputs
- Driver tree with populated values.
- Sensitivity ranking.
- Priority intervention zones (high leverage + high gap).

Deck expression
- Cascade tree with leverage callouts.

Decision unlocked
- Where to intervene to maximize KPI improvement per unit effort.

### Granular Growth Decomposition (GGD)

What it is
- A micro-segment growth decomposition and future potential scoring approach for identifying growth pockets and drag pockets.

Why/when used
- Use for growth strategy, portfolio shifts, market-entry refinement, or slowdown diagnostics.

Inputs
- Multi-dimensional segmentation schema, historical segment performance, forward indicators, right-to-win assessment.

Outputs
- Historical growth attribution by micro-segment.
- Future potential score by segment.
- Resource reallocation recommendations.

Deck expression
- Growth heat map + reallocation flow view.

Decision unlocked
- Where to reallocate budget, capacity, and commercial attention for highest growth return.

## Tier 3: Strategic Lenses

### Where-to-Play / How-to-Win

What it is
- A choice architecture that narrows market arenas and battlegrounds, then defines the capability system required to win.

Why/when used
- Use after core analytics are complete and leadership must choose strategic focus.

Inputs
- Tier 2 evidence, market attractiveness, competitive position, internal capability baseline.

Outputs
- Selected arenas/segments/battlegrounds.
- Winning aspiration per battleground.
- Capability system map and investment implications.

Deck expression
- Cascade of choices + battleground scorecard + capability network.

Decision unlocked
- What to compete in, how to win there, and what capabilities must be built first.

### Horizon Portfolio Refresh

What it is
- A portfolio balancing lens across near-term core, medium-term growth, and longer-term options with migration tracking.

Why/when used
- Use when leadership must rebalance resource allocation and avoid short-term bias.

Inputs
- Initiative list, expected returns, risk profile, timing, strategic fit.

Outputs
- Horizon allocation view.
- Initiative migration tracker over time.

Deck expression
- Horizon portfolio view + migration dashboard.

Decision unlocked
- How much to invest in defend-core versus build-next versus create-option plays.

### Digital Value at Scale

What it is
- A scale-up framework that moves digital/AI efforts from isolated pilots to enterprise value capture.

Why/when used
- Use when many use cases exist but scaled impact is low.

Inputs
- Digital ambition, use-case inventory, data/tech readiness, operating-model constraints.

Outputs
- Prioritized use-case portfolio.
- Enabling architecture blueprint.
- Value tracking cockpit.

Deck expression
- Prioritization matrix + architecture blueprint + cockpit mockup.

Decision unlocked
- Which use cases to scale first and what foundation must be built to realize value.

## Tier 4: Implementation Architecture

### Impact Blueprint

What it is
- A translation layer from strategic recommendations into executable initiatives, sequencing logic, and governance operating model.

Why/when used
- Use when strategy is agreed and execution risk is now dominant.

Inputs
- Strategy choices, quantified targets, org constraints, resource envelope.

Outputs
- Initiative charters.
- Interdependency and sequencing map.
- Transformation office blueprint and cadence.

Deck expression
- Initiative portfolio map + network view + TMO governance page.

Decision unlocked
- What starts when, who is accountable, what is on critical path, and how progress is governed.

## Cross-framework handoff logic

1. Issue Tree defines analytical lanes.
2. Benchmark + Driver + GGD quantify lane economics.
3. Strategic lenses convert economics into directional choices.
4. Impact Blueprint converts choices into execution system.

This handoff logic is the core reason the stack is reusable and skill-convertible later.
