# BCG Framework Catalog

Updated: 2026-03-03
Companion: [process.md](process.md) — Strategic Prism workflow and quality gates.

## Catalog contract

Each framework entry defines:

1. What it is.
2. Why/when used.
3. Inputs.
4. Outputs.
5. Deck expression.
6. Decision unlocked.

## Prism architecture

### Strategic Prism

What it is
- A modular multi-lens architecture that refracts one strategic question into distinct insight vectors and re-converges them into an integrated perspective.

Why/when used
- Use when a single analytical path cannot capture strategic complexity.

Inputs
- Strategic question, context constraints, lens selection policy.

Outputs
- Lens-specific insight outputs and integrated strategic perspective.

Deck expression
- Lens output package with integration summary.

Decision unlocked
- Which strategic direction has strongest cross-lens support.

## Lens 1 frameworks

### Advantage Stack

What it is
- A five-layer model of competitive advantage structure.

Why/when used
- Use to diagnose where advantage exists and where structure is weak.

Inputs
- Competitive data by scale, scope, learning, network, ecosystem dimensions.

Outputs
- Layer-level advantage profile.

Deck expression
- Architecture-style stack diagram.

Decision unlocked
- Which advantage layers require protection, extension, or rebuild.

### Relative Advantage Index (RAI)

What it is
- A relative performance ratio against best competitor on each advantage layer.

Why/when used
- Use to quantify magnitude of advantage/disadvantage.

Inputs
- Client performance and competitor benchmark for each layer.

Outputs
- Layer-level relative index values.

Deck expression
- Indexed score labels within stack/trajectory/gap visuals.

Decision unlocked
- Where parity, leadership, or deficit exists.

### Half-Life Assessment

What it is
- A durability metric estimating time for advantage erosion by 50 percent.

Why/when used
- Use to prioritize durable versus transient advantages.

Inputs
- Replication pace, innovation refresh evidence, structural disruption indicators.

Outputs
- Durability class per layer.

Deck expression
- Color-intensity coding and durability tags.

Decision unlocked
- Which advantages demand immediate reinforcement.

### Advantage Trajectory Map

What it is
- A time-based model of historical and projected advantage evolution.

Why/when used
- Use when static benchmark views are insufficient.

Inputs
- Historical RAI series, projected decay/refresh/disruption assumptions.

Outputs
- Trajectory curves by layer with uncertainty ranges.

Deck expression
- Ribbon chart with archetype labels.

Decision unlocked
- Which advantages are accelerating, eroding, or collapsing.

### Competitive Decay Model

What it is
- A projection model for advantage erosion and refresh under competitive pressure and environmental shifts.

Why/when used
- Use to parameterize forward trajectory assumptions.

Inputs
- Replication speed, refresh rate, exogenous shift probability.

Outputs
- Projected trajectory path and confidence range.

Deck expression
- Trajectory projection overlays.

Decision unlocked
- How quickly reinforcement moves must be executed.

### Strategic Gap Matrix

What it is
- A matrix of current position, required future position, and resulting advantage gap.

Why/when used
- Use after trajectory modeling to quantify strategic gap closure need.

Inputs
- Current RAI, required-future RAI, layer criticality.

Outputs
- Gap severity profile by layer.

Deck expression
- Matrix with critical-gap flags and narrative so-what section.

Decision unlocked
- Which gaps are critical to future competitiveness.

### Strategic Moves Portfolio

What it is
- A portfolio of protect/extend/build/disrupt moves linked to advantage layers.

Why/when used
- Use to operationalize gap closure and advantage creation.

Inputs
- Gap matrix, investment envelope, risk appetite, time horizon.

Outputs
- Balanced move portfolio across horizons and move types.

Deck expression
- Two-axis portfolio map with move bubbles.

Decision unlocked
- Which set of moves should be funded and sequenced.

### Move Card

What it is
- A standardized move definition artifact.

Why/when used
- Use to maintain consistency and uncertainty transparency across move options.

Inputs
- Move design, expected impact, risk and uncertainty assumptions.

Outputs
- Card-level move specification.

Deck expression
- Tiled move-card grid.

Decision unlocked
- Which move is actionable under current uncertainty profile.

## Lens 2 frameworks

### Digital Acceleration Index (DAI)

What it is
- A benchmarked digital maturity diagnostic across strategy, capability, culture, and governance.

Why/when used
- Use to locate digital capability position against external peers.

Inputs
- Digital strategy and operating evidence, capability data, governance evidence, benchmark reference set.

Outputs
- Dimension scores and maturity profile.

Deck expression
- Radar profile vs benchmark median/top quartile.

Decision unlocked
- Which digital dimensions require immediate capability investment.

### AI Impact-Feasibility Matrix

What it is
- A use-case prioritization matrix with impact and feasibility axes plus migration trajectory.

Why/when used
- Use to stage AI use cases by readiness and economic value.

Inputs
- Use-case list, feasibility scoring inputs, impact scoring inputs, 18-month projection assumptions.

Outputs
- Current and projected use-case positions.

Deck expression
- Scatter matrix with current and projected markers connected by arrows.

Decision unlocked
- Which AI use cases to execute now versus prepare for later.

### Digital Value Chains

What it is
- A process-level digital value architecture from data capture to feedback loop.

Why/when used
- Use to identify value-at-stake and digitization gaps along core value chains.

Inputs
- End-to-end process map, digitization status, value potential estimates.

Outputs
- Segment-level digitization and value profile.

Deck expression
- Layered chain map with heat intensity and fill-level indicators.

Decision unlocked
- Where digital and AI investment creates highest value return.

## Lens 3 frameworks

### Smart Simplicity

What it is
- An organizational design framework focused on reducing friction by enabling autonomous cooperation.

Why/when used
- Use when structural complexity blocks strategic execution.

Inputs
- Organizational behaviors, collaboration patterns, decision-rights evidence.

Outputs
- Rule-based diagnosis and intervention priorities.

Deck expression
- Diagnostic package across time allocation, network, and escalation maps.

Decision unlocked
- Which organizational frictions must be removed to execute strategy.

### Shadow Analysis

What it is
- Direct observation method for role-level activity reality.

Why/when used
- Use to identify divergence between formal design and actual behavior.

Inputs
- Role selection and observation logs.

Outputs
- Time Allocation Maps and gap findings.

Deck expression
- Role-level time allocation comparison visuals.

Decision unlocked
- Which roles are overloaded with non-value activities.

### Network Analysis (Integrator Focus)

What it is
- Collaboration network mapping to identify high-betweenness integrators.

Why/when used
- Use to strengthen cross-silo coordination capacity.

Inputs
- Communication and meeting network data.

Outputs
- Integrator map and reinforcement targets.

Deck expression
- Collaboration network map with highlighted integrators.

Decision unlocked
- Which actors should be empowered as integration nodes.

### Decision Rights Audit

What it is
- Escalation-depth analysis by decision type.

Why/when used
- Use to rebalance authority toward faster operational execution.

Inputs
- Decision process traces and escalation paths.

Outputs
- Escalation profile and redesign recommendations.

Deck expression
- Escalation-depth matrix by decision category.

Decision unlocked
- Which decisions should be delegated versus retained.
