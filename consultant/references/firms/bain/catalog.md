# Bain Framework Catalog

Updated: 2026-03-03
Companion: [process.md](process.md) — three-loop spiral workflow and quality gates.

## Catalog contract

Each framework entry defines:

1. What it is.
2. Why/when used.
3. Inputs.
4. Outputs.
5. Deck expression.
6. Decision unlocked.

## Loop 1 frameworks

### Decision Brief

What it is
- A one-page closed-form decision definition for executive choice.

Why/when used
- Use at engagement start to lock decision shape and timeline.

Inputs
- Executive decision requirement, deadlines, stakeholders, evaluation criteria.

Outputs
- Four-field brief: decision, criteria, timeline, stakeholders.

Deck expression
- Four-quadrant opening slide.

Decision unlocked
- What decision is being made, by whom, by when, and using which criteria.

### Option Map with Constraint Mapping

What it is
- A map of feasible and infeasible strategic action options with explicit constraints.

Why/when used
- Use after decision framing to define full action space.

Inputs
- Decision Brief, strategy alternatives, capital and capability constraints.

Outputs
- Option map with financial, organizational, and temporal constraint tags.

Deck expression
- Horizontal option tree with state coding for feasible/infeasible/finalist options.

Decision unlocked
- Which options should advance to deep analysis.

### Rapid Screen Matrix

What it is
- A fast option filter against decision criteria using H/M/L ratings.

Why/when used
- Use to compress broad option sets to a finalist set.

Inputs
- Feasible options, decision criteria, initial judgments.

Outputs
- Screen matrix and finalist list.

Deck expression
- Criteria matrix with H/M/L visual indicators and overall screen profile.

Decision unlocked
- Which options merit full value architecture analysis.

### Conditional Hypothesis Map

What it is
- A condition-based hypothesis statement linking each finalist option to required evidence checks.

Why/when used
- Use before deep analytics to ensure direct decision traceability.

Inputs
- Finalist options, decision criteria, known assumptions.

Outputs
- Conditional hypotheses and mapped validation workstreams.

Deck expression
- Option boxes with condition lists and workstream links.

Decision unlocked
- What must be true for each option to be recommended.

## Loop 2 frameworks

### Bain Value Model

What it is
- A focused multi-scenario economics model for option comparison.

Why/when used
- Use to compare option attractiveness under base/upside/downside states.

Inputs
- Revenue assumptions, cost assumptions, investment profile, hurdle rates.

Outputs
- NPV/IRR/payback/ROIC scenario results by option.

Deck expression
- Scenario comparison charts and economics summary views.

Decision unlocked
- Which options clear return thresholds and how sensitive returns are.

### Competitive Response War Game

What it is
- A simulation of probable competitor responses and economic erosion impact.

Why/when used
- Use when option outcomes depend on competitive reaction.

Inputs
- Competitor profiles, likely response plays, option economics.

Outputs
- Response scenarios and erosion-adjusted downside economics.

Deck expression
- Option-by-competitor response matrix with erosion markers.

Decision unlocked
- Which options are robust under realistic competitor behavior.

### NPS-Linked Value Testing

What it is
- A customer-value testing approach that links JTBD propositions to expected NPS movement and economics.

Why/when used
- Use when customer response is central to option value.

Inputs
- JTBD segmentation, proposition claims, proof points, NPS linkage assumptions.

Outputs
- Segment-level NPS impact estimates and economics implications.

Deck expression
- Customer value maps and NPS-to-economics bridges.

Decision unlocked
- Which options create strongest customer value in priority segments.

### Feasibility Radar

What it is
- A five-dimension implementation readiness assessment.

Why/when used
- Use to evaluate execution risk before recommendation lock.

Inputs
- Structured maturity interviews and benchmark-calibrated scoring.

Outputs
- Talent/technology/process/capital/culture readiness profile.

Deck expression
- Radar comparison chart across finalist options.

Decision unlocked
- Which options are realistically executable at required pace.

### Integrated Option Scorecard

What it is
- A single decision matrix integrating economics, robustness, customer value, and feasibility.

Why/when used
- Use to converge to a recommendation with explicit logic.

Inputs
- Outputs of all Loop 2 modules.

Outputs
- Integrated option ranking using lexicographic decision sequence.

Deck expression
- Matrix scorecard with recommendation emphasis and decision-rule callout.

Decision unlocked
- Which option should be selected and why.

## Loop 3 frameworks

### RAPID Decision Map

What it is
- A governance map of Recommend/Agree/Perform/Input/Decide roles by decision area.

Why/when used
- Use to align implementation governance with organizational decision rights.

Inputs
- Current decision process maps and stakeholder interviews.

Outputs
- Function-level RAPID role assignments.

Deck expression
- RAPID-coded governance map.

Decision unlocked
- Who owns which decision role during execution.

### Result Card

What it is
- A standardized specification of target results and measurement system.

Why/when used
- Use to define execution outcomes before and during sprint delivery.

Inputs
- Target outcomes, baseline metrics, owners, data sources.

Outputs
- Result cards including leading indicators.

Deck expression
- KPI tiles and result-card summary tables.

Decision unlocked
- Which outcomes are on track and where intervention is needed.

### 90-Day Sprint Planning

What it is
- A rolling execution structure that breaks delivery into autonomous 90-day cycles.

Why/when used
- Use to maintain momentum and accelerate value realization.

Inputs
- Result cards, dependency map, capacity constraints.

Outputs
- Sprint objectives, activity plans, ownership, success criteria.

Deck expression
- Sprint-board format with status encoding.

Decision unlocked
- What to execute now, next, and later under constrained capacity.

### Behavioral Change Map

What it is
- A behavior-focused implementation diagnostic and intervention map.

Why/when used
- Use when execution depends on behavior shifts, not only process steps.

Inputs
- Critical behaviors, current-state observations, barrier diagnostics.

Outputs
- Barrier-to-intervention matrix by initiative.

Deck expression
- Initiative-by-barrier matrix.

Decision unlocked
- Which interventions are required to remove adoption friction.

### Results Delivery Office Blueprint

What it is
- A governance and operating blueprint for sustained result delivery.

Why/when used
- Use after option selection and during execution scale-up.

Inputs
- Result cards, sprint design, RAPID mapping, governance requirements.

Outputs
- Office structure, cadence, escalation rules, dashboard blueprint.

Deck expression
- Governance architecture and dashboard mockup.

Decision unlocked
- How progress is governed, escalated, and corrected in-flight.

## Standalone frameworks

### NPS Prism 3.0

What it is
- A three-layer decomposition linking aggregate NPS to episodes, drivers, and economics.

Why/when used
- Use for customer-experience prioritization with quantified value linkage.

Inputs
- Episode survey data, driver metrics, linkage model assumptions.

Outputs
- Episode contribution view, driver impact ranking, economics bridge.

Deck expression
- NPS waterfall, driver impact chart, NPS-to-economics waterfall.

Decision unlocked
- Which customer-experience levers produce highest economic return.

### Dynamic Profit Pool Mapping

What it is
- A time-aware profit pool map with disruptor and adjacency overlays.

Why/when used
- Use for industry structure shifts, adjacency strategy, and migration risk.

Inputs
- Value-chain economics, historical pool data, disruptor dynamics, adjacency definitions.

Outputs
- Historical migration map and forward profit pool projection.

Deck expression
- Two-page spread: historical/current map and projected map.

Decision unlocked
- Which pools are growing, shrinking, or migrating and where to reposition.

### Repeatable Model Diagnostic

What it is
- A diagnostic of repeatability quality across differentiation, non-negotiables, adaptation rules, and replication health.

Why/when used
- Use when scaling a proven model across segments or geographies.

Inputs
- Claimed differentiators, replication practices, adaptation evidence, performance data.

Outputs
- Three-tests differentiation audit, adaptation boundaries, replication health scorecard.

Deck expression
- Differentiation matrix, adaptation spectrum, replication scorecard, roadmap.

Decision unlocked
- How to scale without degrading core differentiation.
