# Firm DNA — Visual Signatures

How each firm's communication style shapes deck structure and layout. For firm epistemology and analytical methodology, see the consultant skill.

## Deck Structure

### Opening Sequence (Slides 1–3)

| Firm | Slide 1 | Slide 2 | Slide 3 |
|---|---|---|---|
| **McKinsey** | Executive Summary (verdict) | Problem Statement (the gap) | Issue Tree Overview |
| **BCG** | Executive Summary (the reframe) | Context / Why Now | Framework Overview (the lens) |
| **Bain** | Decision Brief (the choice) | Options Summary (A vs B vs C) | Evaluation Criteria |

### Closing Sequence (Final 2–3 slides)

| Firm | Penultimate | Final |
|---|---|---|
| **McKinsey** | Implementation Roadmap (phased) | Next Steps + Risk Register |
| **BCG** | Self-Financing Proof (investment pays for itself) | Implications + Strategic Options |
| **Bain** | 90-Day Sprint Plan (what happens Monday) | Decision Request (asks for approval) |

### Transition Style

| Firm | Between-pillar transitions |
|---|---|
| **McKinsey** | Explicit pivot slides. Clean break. "We've established X; now Y." |
| **BCG** | Embedded in section openers. First slide does double duty. |
| **Bain** | Minimal ceremony. Bold action title shifts focus. Decision gates replace narrative transitions. |

## Communication Signatures

### McKinsey — Verdict-First
- Action titles are full-sentence conclusions: "Revenue grew 12% driven by pricing power in mid-tier"
- Analytical machinery lives in the appendix — the main deck is clean narrative
- Waterfall bridges are signature (~30% of analytical slides)
- Exhibit numbering creates formal reference system ("Exhibit 1.2")

### BCG — Framework-First
- The framework IS the product — analytical structure is visible, not hidden
- 2×2 matrices appear 2–3× more than in other firms' decks
- Panel mini-headers on multi-panel slides — BCG never drops these, even at L1
- Self-financing proof: every recommendation must show how it pays for itself

### Bain — Decision-First
- **Facts vs. Perspectives** is the signature composition: left (data), right (interpretation)
- Decks read like court briefs: decision stated on page 1, evidence organized for/against
- 90-day sprint plans close every deck — "what happens Monday"

## Density × Firm

| Firm | L1 Show Deck | L2 Working Deck |
|---|---|---|
| **McKinsey** | 15–20 slides. Zero methodology. Large appendix. | Sensitivity tables enter main deck. Dual-claim titles. |
| **BCG** | Rare. When forced: framework shown but not derived. Still denser than McKinsey L1. | 25–35 slides. Native density. Full annotation layers. |
| **Bain** | Decision brief stays. Minimal transition ceremony. | Facts vs. Perspectives used extensively. |

## Applying Firm Overlay

1. Set palette: `consulting-mckinsey`, `consulting-bcg`, or `consulting-bain`
2. Structure opening and closing sequences per the tables above
3. Apply the communication signature (verdict-first / framework-first / decision-first)
4. Use firm-specific composition rules from SKILL.md § Firm Style
5. Match density preference to the audience
